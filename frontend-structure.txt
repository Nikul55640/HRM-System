# HRM System Frontend Structure Documentation

## Overview
This document provides a comprehensive overview of the HRM System frontend architecture, built with React, Vite, and modern JavaScript/JSX.

## Key Technologies & Libraries

### Core Framework
- **React 18**: Modern React with hooks and concurrent features
- **Vite**: Fast build tool and development server
- **JavaScript/JSX**: Primary development language

### UI & Styling
- **Tailwind CSS**: Utility-first CSS framework
- **Shadcn/UI**: Pre-built accessible components
- **Lucide React**: Icon library
- **React Hot Toast**: Toast notifications

### State Management
- **Zustand**: Lightweight state management
- **React Hook Form**: Form state management
- **Zod**: Schema validation

### Routing & Navigation
- **React Router DOM**: Client-side routing
- **Protected Routes**: Role-based access control

### HTTP & API
- **Axios**: HTTP client for API calls
- **React Query**: Server state management (if used)

### Development Tools
- **ESLint**: Code linting
- **Jest**: Testing framework
- **Babel**: JavaScript transpilation
- **PostCSS**: CSS processing

### Date & Time
- **date-fns**: Date manipulation library
- **React Day Picker**: Date picker component

### Forms & Validation
- **Formik**: Form library
- **Yup**: Schema validation
- **React Hook Form**: Modern form handling

## Architecture Patterns

### 1. Feature-Based Organization
- Each major feature (attendance, leave, employees) has its own module
- Modules contain pages, components, services, and stores
- Promotes code organization and maintainability

### 2. Layered Architecture
- **Core**: Shared utilities, authentication, layout
- **Modules**: Feature-specific code
- **Shared**: Reusable components and UI elements
- **Services**: API communication layer

### 3. Component Composition
- Atomic design principles with shared UI components
- Higher-order components for common functionality
- Compound components for complex UI patterns

### 4. State Management Strategy
- Zustand for global application state
- Local state for component-specific data
- Form state managed by React Hook Form

### 5. Route Protection
- Role-based access control
- Permission gates for fine-grained access
- Protected route wrappers

## Development Workflow

### 1. Environment Setup
```bash
npm install          # Install dependencies
npm run dev         # Start development server
npm run build       # Build for production
npm run preview     # Preview production build
npm test            # Run tests
npm run lint        # Run ESLint
```

### 2. Component Development
- Use Shadcn/UI components as base
- Follow atomic design principles
- Implement proper TypeScript/PropTypes
- Add proper error boundaries

### 3. API Integration
- Use service layer for API calls
- Implement proper error handling
- Add loading states and optimistic updates
- Cache responses where appropriate

### 4. Testing Strategy
- Unit tests for utilities and hooks
- Component tests for UI components
- Integration tests for user workflows
- E2E tests for critical paths
